The data flow

How it all works, step by step

The v1 happy path — each step tagged with the system doing the work.

1
Catalog & rules sync inHubSpotCustom objApp
product records (with cpq_* flags) and product_association rules sync into the app. This is what turns a flat product list into a configurator.
2
Rep builds the dealApp
Embedded on the HubSpot deal, the rep adds one or more machines → the rules surface only valid options → guided selling. Saves to quote_configurations, snapshots to quote_versions.
3
Pricing & leasing resolveApp
pricing_rules / tiers set markup off floor cost; lease_rate_sheets compute the monthly payment; blended margin recalcs against target.
4
Proposal generatesApp
document_templates + the configuration produce one consistent, branded proposal — the exact gap VanAusdall named.
5
Push back to the DealApp→ HubSpot
A clean set of line_items grouped by machine (machine_group + line_role — line-item associations are read-only in HubSpot), plus deal.amount, margin, and lease fields. HubSpot is now the record.
6
Sign & closeHubSpot
HubSpot Quote / PandaDoc / DocuSign e-sign → accepted → workflow advances the stage and fires downstream automation.

Build path

Most of the engine already exists. The configurator and its rules are the real build.

In Marko's app: OAuth + read/write, line-item writeback, quote versions & templates, leasing engine, pricing tiers, document generation, field mappings, roles, multi-tenant settings.

Standardize cpq_* flags + rep/dealer cost on the price book; add pricing_rules (markup criteria, effective dates).

Rule fields on product_association + the guided-selling configurator UI and its rules engine. The core of v1.
